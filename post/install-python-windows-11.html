How to Install Python on Windows 11 (Step-by-Step 2026)

A complete guide to installing Python 3 on Windows 11, including PATH configuration, package management verification, and running your first script all in about five minutes.

July 12, 2026 Melalew Mengistu

A junior developer messaged up their PATH variable during onboarding. Every time they opened a terminal, the system couldn't find Python. They spent an hour trying to figure out why python returned "'python' is not recognized as an internal or external command." The fix took thirty seconds: reinstall with the checkbox checked.

This is the single most common Python installation mistake on Windows. The installer handles PATH correctly if you let it but if you miss that checkbox, you'll spend more time troubleshooting than you spent installing.

What PATH Actually Does

PATH is a system variable that tells Windows which folders to search when you type a command in the terminal. When you type python, Windows checks each folder in the PATH list from left to right until it finds python.exe. If the Python folder isn't in that list, you get "'python' is not recognized."

Before Python 3.4, this checkbox wasn't checked by default and users had to add the path manually. Now it is, which is why the installation is straightforward but only if you don't skip that step.

There's a second reason PATH matters: if you install multiple Python versions, the last one installed becomes the default. The previous version is still accessible by its full path like py -3.11 instead of python.

What the Installer Gives You
  • Python interpreter The actual language runtime that executes .py files. This is what you're really installing; everything else is bonus.
  • pip Python's package manager. Lets you install libraries like Django, NumPy, or Requests from PyPI with a single command.
  • IDLE A lightweight graphical editor and REPL bundled with Python. Useful for quick experiments, though most developers switch to VS Code or PyCharm.
  • py launcher A Windows-specific tool that lets you pick which Python version runs when you double-click a .py file or type py in the terminal.

Common Errors and Fixes

ERROR 'python' is not recognized as an internal or external command, operable program or batch file.
FIX

Cause: The Python installation folder is not in your system PATH. This happens when the "Add python.exe to PATH" checkbox was unchecked during installation.

Fix: Reinstall Python and make sure the checkbox is checked. Alternatively, manually add the path:

# Add these two paths to System PATH:
C:\Users\YourName\AppData\Local\Programs\Python\Python313\
C:\Users\YourName\AppData\Local\Programs\Python\Python313\Scripts\

Search "environment variables" → "Edit the system environment variables" → "Environment Variables" → Find "Path" under User variables → Edit → Add both paths → Restart your terminal.

ERROR 'pip' is not recognized as an internal or external command, operable program or batch file.
FIX

Cause: The Scripts\ subfolder inside your Python installation isn't in PATH. This folder contains pip.exe. The main Python folder may be in PATH, but Scripts often gets missed in manual setup.

Fix: Add the Scripts folder to PATH (same method as Error 1), or use this shortcut that works regardless of PATH:

python -m pip --version

If that works, your pip is fine it's just the Scripts folder missing from PATH. Add it and pip will work directly.

ERROR SSLCertVerificationError(1, '[SSL: CERTIFICATE_VERIFY_FAILED] certificate verify failed: unable to get local issuer certificate')
FIX

Cause: Corporate proxies, antivirus SSL inspection, or outdated certificate bundles can break pip's HTTPS connections to PyPI.

Option A: update certificates:

python -m pip install --upgrade certifi
python -m pip install --trusted-host pypi.org --trusted-host files.pythonhosted.org package_name

Option B: if behind a proxy: Configure your proxy settings or ask IT to add pypi.org and files.pythonhosted.org to the SSL inspection allowlist.

ERROR Could not install packages due to an OSError: [Errno 13] Permission denied: 'C:\Program Files\Python313\Lib\site-packages\...'
FIX

Cause: Python was installed for "All Users" (to Program Files) and you don't have write permission without admin rights.

Option A (recommended):

pip install --user package_name

Option B: Run Command Prompt as Administrator and retry.

Option C (best long-term): Reinstall Python with "Install just for me" instead of "Install for all users" to avoid permission issues entirely.

ERROR Python 3.13.2 Setup failed with error code 0x80070002. The system cannot find the file specified.
FIX

Cause: The installer file is corrupted (incomplete download) or your antivirus quarantined part of it during execution.

Fix:

# 1. Re-download the installer (don't reuse the old file)
# 2. Temporarily disable real-time protection in Windows Defender
# 3. Right-click the installer → Properties → check "Unblock" at bottom
# 4. Run the installer again

If it still fails, try the web-based installer instead of the offline one — it's a smaller download that pulls components on demand and is less likely to trigger antivirus heuristics.

ERROR python --version → Python 3.11.5 (expected 3.13)
FIX

Cause: When multiple Python versions are installed, the one whose folder appears first in PATH wins.

Fix: Use the py launcher to explicitly select a version:

py -3.13 --version    # Explicitly use 3.13
py -3.11 --version    # Explicitly use 3.11
py --list             # Show all installed versions

To change the default, reorder the Python folders in your PATH variable so the preferred version comes first, or use py as your default command instead of python.

What We'll Do

Five minutes of actual work, split into three phases: download, install with correct configuration, and verification.

1 Download the correct installer from python.org and choose the right version.
2 Install with PATH enabled and understand what the installer is doing.
3 Verify Python, pip, and run your first script to confirm everything works end-to-end.

Step 1

Download the Right Installer

Third-party download sites bundle Python with adware or modified installers. The official source is the only one that guarantees an unmodified, cryptographically verified binary. Always download from python.org.

1

Go to python.org/downloads

Navigate to python.org/downloads. The page detects your operating system automatically and recommends the correct installer.

2

Choose the right version

Download the latest stable release highlighted at the top of the page (e.g., Python 3.12.x). Click the yellow "Download Python 3.12.x" button. Don't download Python 2 it reached end of life in 2020 and is no longer maintained.

3

Choose 64-bit

On the download page, make sure "Windows installer (64-bit)" is selected, not the 32-bit version. All modern Windows 11 systems are 64-bit. The 64-bit version can access more than 4GB of RAM, which matters for data science, machine learning, and running large applications. The only reason to choose 32-bit is if you're working with legacy enterprise software that doesn't have a 64-bit version available.

Windows Store version: Windows 11 also offers Python through the Microsoft Store. Avoid it for development work. The Store version has limitations it installs to a hidden AppData folder, doesn't include pip by default, and makes PATH configuration harder. The official installer from python.org gives you a standard, full-featured installation.

Step 2

Install with PATH Enabled

The PATH checkbox tells Windows where Python lives so you can run it from any terminal. Without it, every terminal command fails with "'python' is not recognized." This is the single most important setting in the entire installation.

1

Run the installer

Find the downloaded .exe in your Downloads folder and double-click it. Windows may show a "User Account Control" prompt click "Yes."

2

Check the PATH checkbox

At the the bottom of the first installer screen, you'll see a checkbox labeled "Add python.exe to PATH". Make sure it's checked. This is checked by default in modern Python installers, but verify it anyway. Click "Install Now."

3

Choose "Install Now" over "Customize installation"

"Install Now" gives you a standard setup with pip, IDLE, and the py launcher. "Customize installation" is useful if you need to exclude specific components, change the install location, or opt out of pip. For a first-time setup, "Install Now" is the right choice.

4

Wait for completion

The installer takes about 30–60 seconds. Let it finish. You'll see a "Setup was successful" message when done.

Disable PATH during installation if you already have another Python version. If you're installing a second Python version alongside an existing one, uncheck "Add to PATH" during installation. You can add the new version to PATH manually afterward to avoid breaking tools that depend on the current default version.

Step 3

Verify Python, pip, and IDLE

Installation appearing successful doesn't guarantee it actually works. Verification takes about one minute but catches PATH issues, broken pip installations, and silent failures that the installer won't tell you about.

1

Check Python version

Open Command Prompt (cmd in the Start menu, or press Win + R and type cmd. Run:

python --version

You should see output like Python 3.12.x. If you see an error about Python not being recognized, the PATH wasn't set correctly. Go back to Step 2.

2

Check pip version

Pip is Python's package manager. Verify it's working:

pip --version

You should see version, version, and location information. If you see a deprecation warning about pip version, run python -m pip install --upgrade pip to update it.

3

Check IDLE

Open the Start menu, type IDLE, and open it. IDLE is Python's built-in editor with syntax highlighting and an interactive shell. You don't need it for professional work most developers use VS Code but it's useful for quick tests and learning.

Run pip list to see all installed packages. This shows everything pip has installed globally on your system. If you see unexpected packages you don't remember installing, it may indicate a permissions issue or another user installed something system-wide.

Step 4

Run Your First Python Script

Seeing a successful version number proves the installation is technically correct, but actually running code confirms it works end-to-end. Create a simple test script to validate everything from the interpreter to the runtime.

1

Create a Python file

Open Notepad or any text editor. Paste the following:

print("Hello, World!")

Save it as hello.py in your Documents folder.

2

Run it from terminal

Open Command Prompt, navigate to your Documents folder, and run:

python hello.py

You should see Hello, World! printed in the terminal. If you do, your Python installation is fully functional.

3

Try installing a real package

Let's install requests, one of the most widely used Python libraries for making HTTP requests:

pip install requests

Then open Python and run:

import requests
response = requests.get("https://http://example.com")
print(response.status_code)

If you see 200 printed, the entire chain works: Python runs, pip installs from PyPI, the library works, and network access functions correctly.

Python environment is ready. You have a working interpreter, a package manager, and a verified runtime. Everything a beginner needs to start writing Python code is now in place.

Bonus

Set Up a Code Editor

IDLE works for quick tests, but it lacks the features professional developers rely on: autocomplete, linting, debugging, and integration with version control systems.

1

Install VS Code

Download VS Code (free, open-source). During installation, it will automatically detect your Python installation and suggest the official Python extension.

2

Install the Python extension

VS Code will suggest installing the Python extension by Microsoft. Install it. This gives you syntax highlighting, code completion, linting, debugging, and the ability to run Python code directly inside the editor with a terminal at the bottom.

3

Create a virtual environment

In VS Code, open the terminal and run:

python -m venv myproject

Activate it with:

myproject\Scripts\activate

Virtual environments isolate project dependencies so different projects don't conflict with each other. This is standard practice in professional Python development and something you should learn early.

Troubleshooting

Common Errors and Fixes

Most installation problems trace back to one of these root causes. Fix the root cause rather than trying random fixes.

"'python' is not recognized as an internal or external command"

This means the Python folder isn't in your PATH. Fix it by finding the Python installation folder (usually C:\Users\YourName\AppData\Local\Programs\Python\Python312\), copying that path, and adding it to your system PATH variable. Then reopen Command Prompt.

"pip" is not recognized

This means pip wasn't installed or isn't in PATH. Run python -m ensurepip to bootstrap pip, then retry. If that fails, reinstall Python with PATH checked. Verify pip with pip --version.

"pip install" fails with a long red error

This usually means your network is blocking PyPI connections (common in corporate environments), or you're behind a proxy that interferes with SSL. Try pip install --trusted-host pypi.org to bypass SSL verification. If you're on a corporate network, you may need to configure proxy settings in pip.ini or ask your IT department for an exception.

Two Python versions are installed but the wrong one runs

This happens when you install a new Python version with "Add to PATH" still checked — the new version overwrites the default python command. The old version is still installed and accessible by its version number, like py -3.11. To control which version is default, open System Environment Variables, find Python in the PATH list, and reorder the entries so your preferred version comes first.

Scripts work in IDLE but fail in terminal

IDLE uses its own Python environment that may differ from your system Python. If pip list shows packages in IDLE but python script.py fails in Command Prompt, IDLE is using a different Python environment. Open a regular Command Prompt and try the command there instead.
