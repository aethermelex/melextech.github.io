How to Install Kali Linux Dual Boot (No Errors)

A step-by-step guide to installing Kali Linux alongside Windows 11 on a UEFI system. Covers partitioning, boot configuration, and the common errors that derail most first-time installations.

July 12, 2026 Melalew Mengistu
What Dual Booting Actually Does

Dual booting means installing two operating systems on the same drive, each in its own partition. When you power on the PC, a boot menu (GRUB) lets you choose which OS to load. Your Windows partition and your Kali partition are completely separate files on one side don't touch the other.

The process involves three things that can go wrong: getting the partition table wrong, misconfiguring the UEFI boot entries, or having Windows Fast Startup lock the drive. This guide addresses all three upfront so you don't hit them during installation.

This guide assumes a modern Windows 10/11 PC with UEFI firmware and a GPT partition table which covers virtually every PC sold since 2018.

Process Overview

We'll prepare Windows first (because it's the hardest to fix after the fact), then create the bootable USB and run the Kali installer.

1 Download ISO and flash a bootable USB with correct partition scheme.
2 Shrink Windows partition and disable features that block Linux installation.
3 Boot from USB, install Kali into the free space, and verify GRUB works.

Step 1

Download Kali and Create Bootable USB

The installer needs to run from external media. The USB must be flashed in GPT mode with a UEFI-compatible bootloader, otherwise your PC won't recognize it as a bootable device.

1

Download the ISO

Go to the official Kali downloads page and grab the 64-bit Installer image. Don't download the "Live" version for a dual-boot install the installer version gives you a persistent installation with proper partition management. The file will be around 4GB.

2

Flash with Rufus

Download Rufus (free, no installation required). Insert an 8GB+ USB drive. In Rufus, select your USB, select the Kali ISO, and critically: set Partition scheme to GPT and Target system to UEFI (non-CSM). Leave everything else default and click Start. Rufus will format the drive and write the bootable image.

Alternative on Linux or Mac: Use balenaEtcher or the dd command. The key requirement is the same the USB must be GPT-partitioned with an EFI system partition, which Etcher handles automatically.

Step 2

Shrink Windows Partition

Kali needs unallocated disk space to create its root, home, and swap partitions. You create this by shrinking your existing Windows partition not by deleting it or creating a new volume in Windows (which would format it as NTFS, and Kali can't use that).

1

Open Disk Management

Right-click the Start button and select Disk Management. You'll see your drive layout typically one large C: partition taking up the entire disk.

2

Shrink the volume

Right-click the C: drive, select Shrink Volume. Windows will query for available shrink space. Enter at least 40960 MB (40GB). Click Shrink. You'll see a new black bar labeled "Unallocated" appear next to C:.

3

Don't create a new volume here

Windows will offer to create a new simple volume from the unallocated space. Skip this. The Kali installer needs raw unallocated space if you format it as NTFS or exFAT in Windows, the installer won't be able to use it properly for Linux partitions.

How much space? 40GB is the practical minimum for Kali if you plan to install tools and store files. 60–80GB is comfortable. If you have a 256GB drive, 40GB is fine. If you have a 1TB drive, allocate more you'll thank yourself later when you're working with large datasets or VM images.

Step 3

Disable Fast Startup and Configure UEFI

Two Windows features commonly break dual-boot setups. Fast Startup writes a hibernation-like file that locks the NTFS partition, preventing Linux from reading or modifying it. Secure Boot can block the Kali bootloader if it's not signed. Both need to be handled before you boot the USB.

1

Disable Fast Startup in Windows

Open Control Panel → Power Options → click "Choose what the power buttons do" on the left. Click the blue text "Change settings that are currently unavailable." Uncheck "Turn on fast startup (recommended)". Save changes.

2

Enter BIOS/UEFI

Restart your PC. During the boot logo screen, press the BIOS key repeatedly. This varies by manufacturer: F2 for Dell/ASUS, F10 for HP, Delete for most custom boards. If you miss it, hold Shift + Restart from Windows → Recovery → Advanced Startup → UEFI Firmware Settings.

3

Disable Secure Boot

Navigate to the Boot or Security tab. Set Secure Boot to Disabled. While Kali technically supports Secure Boot with signed shim, disabling it eliminates a whole category of boot failures on first install. You can re-enable it later after GRUB is properly configured.

4

Verify Boot Mode is UEFI

Still in BIOS, confirm "Boot Mode" is set to UEFI, not "Legacy" or "CSM." If you see a "CSM" (Compatibility Support Module) option, disable it. Save changes and exit. The PC will reboot.

BitLocker warning: If your Windows drive is BitLocker-encrypted, shrinking the partition may trigger a BitLocker recovery on next boot. Make sure you have your recovery key saved (usually in your Microsoft account). The shrink operation itself doesn't encrypt or decrypt anything it just resizes the partition but some BIOS changes can trigger BitLocker's tamper detection.

Step 4

Boot from USB and Install Kali

With the preparation done, the actual installation is straightforward. The key decision point is the partitioning step selecting the wrong option here is how people accidentally overwrite Windows.

1

Boot from the USB

Insert the USB. Reboot and press the boot menu key (F12, F11, or Esc depending on your motherboard). Select the USB drive from the list. You'll see the Kali boot menu select the first option (Graphical Install).

2

Walk through the installer prompts

Select your language, location, and keyboard layout. When you reach the network configuration step, connect to Wi-Fi or plug in Ethernet the installer needs network access to fetch package lists and install software.

3

Partitioning the critical step

When you reach "Partition disks," select "Guided use the largest continuous free space." This is the option that targets only the unallocated space you created. Do NOT select "Guided use entire disk" that will wipe Windows. The installer will automatically create / (root), /home, and swap partitions within the free space.

4

Finish the installation

Set your username and password. Select "Install the GRUB boot loader to the master boot record" it should default to your primary drive (e.g., /dev/nvme0n1), not a partition like /dev/nvme0n1p1. Complete the install. When prompted, remove the USB and reboot.

GRUB target matters: If GRUB installs to a partition instead of the drive, Windows Boot Manager won't chainload it and you'll boot straight to Windows with no GRUB menu. If this happens, you'll need to boot from the Kali Live USB and reinstall GRUB using grub-install /dev/nvme0n1 from a chroot environment.

Step 5

Post-Installation Verification

After the first boot into Kali, a few things need checking before you start using it for real work.

1

Verify the GRUB menu appears

Reboot the PC. You should see the GRUB menu with both Kali Linux and Windows Boot Manager listed. Select Windows to confirm it still boots. If GRUB doesn't appear, see the FAQ section below.

2

Update Kali immediately

Open a terminal and run sudo apt update && sudo apt full-upgrade -y. The ISO you downloaded may be weeks old, and Kali's tools update frequently. Running this before you start working ensures you have current package versions and security patches.

3

Verify Windows is accessible from Kali

Kali should auto-mount your Windows partition under /mnt/windows or a similar path. Open your file manager and check. If it's not mounted, you can mount it manually with sudo mount -t ntfs-3g /dev/nvme0n1p3 /mnt/windows (adjust the partition number to match your Windows partition).
